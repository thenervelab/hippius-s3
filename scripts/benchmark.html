Throughput benchmarks for our two supported regions: eu-central-1.hippius.com (also reachable as s3.hippius.com) and us-east-1.hippius.com.

  • eu-central-1.hippius.com
  • us-east-1.hippius.com

Upload throughput

How fast clients can upload a 2 GB object to each region.

Upload latency

End-to-end time to PUT a 2 GB object.

Download throughput

Typical download — content served from the regional cache.

Download TTFB

Time to first byte for typical downloads served from the regional cache.

Uncached download throughput

First-time download of a fresh object — content fetched from origin storage.

Uncached download TTFB

Time to first byte on first-time downloads.

Benchmark runtime

Active S3 time per run — first upload to last download, excluding file generation and setup.

All benchmarks use 2 GB test files via a single PUT request (no multipart upload). Each run records 3 trials. All timestamps are in UTC. Equivalent AWS CLI invocations: 
# Upload — single PUT, no multipart
aws s3api put-object --bucket BUCKET --key KEY --body ./file.bin \
  --endpoint-url https://eu-central-1.hippius.com

# Download
aws s3 cp s3://BUCKET/KEY ./file.bin \
  --endpoint-url https://eu-central-1.hippius.com

Source: scripts/s3-benchmark.py in the hippius-s3 repo.
