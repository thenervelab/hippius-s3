# tests/integration/

Tests that exercise real DB + Redis via docker-compose, but mock external services (Arion, KMS, Hippius API). In-between unit (everything mocked) and e2e (everything real).

## Scope

- Worker logic with real queue + DB state transitions.
- Cache behavior against real Redis pub/sub.
- ACL evaluation against real Postgres.
- Some cross-module contract tests (writer → reader round trips).

## Prereqs

```bash
docker compose up -d postgres redis redis-accounts redis-chain redis-queues redis-rate-limiting redis-acl
```

(The full stack isn't needed — just the data plane.)

Then:

```bash
pytest tests/integration -v
```

Env: [.env.test-local](../../.env.test-local) (localhost connections).

## Conventions

- **Use `HIPPIUS_BYPASS_CREDIT_CHECK=true`** and `ENABLE_BANHAMMER=false` — set in [conftest.py](conftest.py).
- **Fresh DB state per test**: uses a pytest fixture that truncates tables between tests, or sets up a schema migration inside a transaction that's rolled back. Don't leave cross-test state.
- **boto3 client fixture** ([conftest.py:46-72](conftest.py)): mocks the Arion endpoint at localhost:8000. Checks `is_real_aws()` to optionally swap for real AWS if `RUN_REAL_AWS=1` is set.
- **Fixed seed phrase** (line 42 of conftest) for deterministic SigV4.

## What's in here

- Presigned URL tests (boto3 generates, client POSTs).
- ACL scope integration (mocks ACL service grants, not the scope evaluator — which is dormant; see [../../gateway/services/CLAUDE.md](../../gateway/services/CLAUDE.md)).
- Migration smoke tests — v4→v5 with fixtures.
- Worker queue round trips (enqueue → worker picks up → DB row updated).

## Running

```bash
pytest tests/integration -v
pytest tests/integration/test_presigned.py::test_presigned_put -xvs
```

## Gotchas

- Tests rely on `docker compose` services being up. If a test hangs on connection, check `docker compose ps`.
- **Mocks sometimes pretend to be master tokens**: [mock-hippius-api] hardcodes `token_type="master"` — if you're testing sub-token behavior, that test belongs in unit (not here) or needs a fresh mock.
- **Redis pub/sub tests are flaky on slow CI** — give them generous timeouts, don't race-condition them.
