# hippius_s3/dlq/

Dead-letter queue for operations that permanently failed. Generic base + two concrete subclasses (upload, unpin). Backed by Redis LPUSH onto dedicated DLQ lists — not the filesystem.

## Files

| File | Purpose |
|---|---|
| [base.py](base.py) | `BaseDLQManager[T]` — generic DLQ base with per-identifier locking, peek/push/requeue/purge/export/import. |
| [upload_dlq.py](upload_dlq.py) | Upload-specific subclass: identifier is `(object_id, upload_id)`, enqueues back to `{backend}_upload_requests`. |
| [unpin_dlq.py](unpin_dlq.py) | Unpin-specific subclass: identifier is the chunk_backend row, enqueues back to `unpin_requests`. |

## On-Redis layout

Each DLQ is a Redis list:

- Upload DLQ key: `{backend}_upload_requests:dlq` (e.g. `arion_upload_requests:dlq`).
- Unpin DLQ key: `unpin_requests:dlq`.

Entries are JSON blobs pushed via `LPUSH` ([base.py:64](base.py)):

```json
{
  "payload": { ... model_dump() of the original request ... },
  "attempts": <int>,
  "first_enqueued_at": <float seconds>,
  "last_attempt_at": <float seconds>,
  "last_error": "<str>",
  "error_type": "transient" | "permanent"
}
```

## Per-identifier locks

`BaseDLQManager._acquire_lock(identifier, ttl_ms=60000)` ([base.py:44](base.py)) uses `SET NX PX` on `dlq:requeue:lock:{dlq_key}:{identifier}`. Prevents concurrent requeues of the same entry. Released atomically via a Lua script comparing the token ([base.py:50](base.py)).

## Requeue flow

[base.py:117 `requeue(identifier)`](base.py):

1. Acquire per-identifier lock (returns False if another requeue is in progress).
2. `_find_and_remove_entry(identifier)` — subclass-specific.
3. If `error_type == "permanent"` and `force=False`, push it back and refuse.
4. Validate the payload via `request_class.model_validate` (pydantic).
5. Reset `attempts = 0` unless `force=True`.
6. Set `bypass_billing = True` if requested (and the payload supports it).
7. `enqueue_func(payload)` — pushes back onto the primary queue.

`requeue_all(force=False, batch_size=500, bypass_billing=False)` ([base.py:160](base.py)) is the bulk version used by [../scripts/dlq_requeue.py](../scripts/dlq_requeue.py). Uses pipelined `rpop` for efficiency; permanent errors get pushed back with `lpush` unless `force=True`.

## Stats / export / import

- `stats()` → `{total_entries, error_types: {transient, permanent, unknown}}`.
- `peek(limit=10)` → last N entries without removing.
- `export(file_path)` → dump to JSON.
- `import_dlq(file_path)` → restore from JSON (LPUSH each entry).

Used by the CLI at [../scripts/dlq_requeue.py](../scripts/dlq_requeue.py) to inspect and selectively requeue.

## Janitor protection

The janitor scans DLQs to protect object_ids from cache eviction while the DLQ entry is still pending. See [workers/run_janitor_in_loop.py:220 `get_all_dlq_object_ids`](../../workers/run_janitor_in_loop.py). The list of DLQs is derived from `config.upload_backends` dynamically — adding a new backend automatically extends protection.

## Adding a new DLQ

1. Subclass `BaseDLQManager[YourRequestType]`.
2. Implement `_get_identifier(payload) -> str` — a unique key for locking + lookup.
3. Implement `_find_and_remove_entry(identifier)` — scan the list and remove the matching entry.
4. Implement `_extract_identifier_from_entry(entry)` — used by `requeue_all`.
5. Optionally override `_create_entry` to add worker-specific fields.
6. Optionally override `_bulk_enqueue` for pipelined primary-queue insert.
